Thermostat using STM32
This project implements a simple embedded thermostat system using an STM32 microcontroller. The system reads temperature from a DS18B20 sensor, displays operating information on an LCD1602, allows the user to adjust the desired setpoint using push buttons, and controls a cooling fan with a basic ON/OFF algorithm.
The project was developed for the Embedded System Programming course project and was verified using Proteus simulation.
---
Main Features
Reads temperature from a DS18B20 digital temperature sensor.
Requests a new temperature sample every 500 ms.
Displays temperature with 1°C resolution.
Shows current temperature, setpoint, operating state, and fan status on an LCD1602.
Supports button-based user interaction:
`SLEEP`: switch between Sleep mode and Normal mode.
`SET`: enter or confirm Setpoint mode.
`UP`: increase setpoint.
`DOWN`: decrease setpoint.
Limits the setpoint range from 0°C to 70°C.
Uses a simple ON/OFF fan control algorithm.
Forces the fan OFF during initialization, Sleep mode, and invalid sensor conditions.
Uses a modular software structure with Application, BSP, and Component Driver layers.

Application Flow
The application is initialized by `app_main_init()`:
Initialize the delay module.
Initialize the button, LCD, and fan BSP modules.
Set the initial state to `S_THERMO_SLEEP`.
Set the default setpoint to 28°C.
Turn the fan OFF for safety.
Initialize the DS18B20 sensor.
Request the first temperature sample if the sensor is ready.
Update the LCD for the first time.
After initialization, the main loop repeatedly calls `app_main_fsm()`.
```c
void app_main_fsm(void) {
    uint8_t button_event = bsp_button_get_events();

    app_button_handle(button_event);
    app_sensor_handle();
    app_fan_handle();
    app_lcd_handle();
}
```

Thermostat State Machine
The application uses three main states:
State	Description	Fan Behavior
`S_THERMO_SLEEP`	Monitor-only mode	Fan is forced OFF
`S_THERMO_NORMAL`	Normal automatic thermostat operation	Fan depends on temperature and setpoint
`S_THERMO_SETPOINT`	User edits the setpoint value	Fan control is still active because the system is not in Sleep mode
State Transitions
Current State	Event	Next State	Action
`S_THERMO_SLEEP`	`SLEEP`	`S_THERMO_NORMAL`	Enable normal thermostat operation
`S_THERMO_NORMAL`	`SLEEP`	`S_THERMO_SLEEP`	Turn fan OFF
`S_THERMO_NORMAL`	`SET`	`S_THERMO_SETPOINT`	Copy current setpoint to editable setpoint
`S_THERMO_SETPOINT`	`UP`	`S_THERMO_SETPOINT`	Increase editable setpoint if below maximum limit
`S_THERMO_SETPOINT`	`DOWN`	`S_THERMO_SETPOINT`	Decrease editable setpoint if above minimum limit
`S_THERMO_SETPOINT`	`SET`	`S_THERMO_NORMAL`	Save edited setpoint
Undefined state	Undefined	`S_THERMO_SLEEP`	Return to safe state and turn fan OFF

Fan Control Algorithm
The fan uses a simple ON/OFF control rule:
```text
Fan ON  if current temperature > setpoint
Fan OFF if current temperature <= setpoint
```
In the implementation, the fan only turns ON when all of these conditions are true:
```c
if ((app_state != S_THERMO_SLEEP) &&
    (current_temp_valid != 0U) &&
    (current_temp_c > setpoint_c)) {
    fan_should_on = 1U;
}
```
This means the fan is always OFF when:
The system is in Sleep mode.
The temperature value is invalid.
The current temperature is lower than or equal to the setpoint.

Timing Design
The application requests a new temperature sample every 500 ms:
```c
#define APP_TEMP_READ_PERIOD_MS 500U
```
The timing check is performed using the delay module:
```c
if (DELAY_IsExpired(temp_read_tick, APP_TEMP_READ_PERIOD_MS) == 0U) {
    return;
}
```
The DS18B20 driver is processed in a non-blocking style. The application requests a conversion, continues running other tasks, and later checks whether new temperature data is ready.

Button Handling
The button BSP uses event flags and debounce checking.
Event definitions:
```c
#define BSP_BUTTON_EVENT_NONE  0U
#define BSP_BUTTON_EVENT_SLEEP (1U << 0)
#define BSP_BUTTON_EVENT_UP    (1U << 1)
#define BSP_BUTTON_EVENT_DOWN  (1U << 2)
#define BSP_BUTTON_EVENT_SET   (1U << 3)
```
Debounce time:
```c
#define BSP_BUTTON_DEBOUNCE_MS 100U
```
After the application reads the pending button events through `bsp_button_get_events()`, the event flags are cleared automatically.

Limitations and Future Improvements
Current limitations:
The system has been verified mainly in Proteus simulation.
The fan control uses simple ON/OFF control without hysteresis.
The setpoint is not stored in non-volatile memory, so it returns to the default value after reset or power loss.
The fan is controlled as a simple digital output, not by PWM speed control.
Possible improvements:
Add hysteresis to reduce frequent ON/OFF switching around the setpoint.
Store the user-defined setpoint in Flash or EEPROM.
Add sensor error messages on the LCD.
Add UART logging for debugging.
Add PWM fan speed control.
Verify the design on real hardware.
